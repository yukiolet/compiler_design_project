# 编译原理课程设计

## 项目选择

第三步选择：
- 3.2 实现中间代码解释器

第四步选择：
- 4.2 四元式到 LLVM IR 的简化转换器
- 4.4 编译前端局部优化与流图分析系统

## 开发语言

Python

## 项目目标

本项目基于已有类 C 语言编译前端和四元式生成结果，继续实现：

1. 四元式解释执行；
2. 四元式到 LLVM IR 风格代码转换；
3. 基本块划分；
4. CFG 控制流图构建；
5. DAG 局部优化；
6. 优化前后四元式对比；
7. 简单可视化展示。

## 注意事项

1. 不要一开始重写词法分析、语法分析和语义分析。
2. 优先围绕已有四元式文件进行后端开发。
3. 新代码应放在 modules/ 或 src/ 文件夹中。
4. old_code/ 文件夹中的代码作为参考，不要直接覆盖。
5. 输出结果保存到 output/ 文件夹中。
